## RayMol 1.4.1

A stability and polish release on top of 1.4.0 — fewer crashes, lighter memory use, and smoother editing.

### Stability
- **Fixed several crashes and a data-loss case.** Three issues found in review that could crash the app or lose unsaved work are resolved.
- **Plugged Metal memory & resource leaks.** The renderer no longer accumulates GPU memory and resources over a long session, so RayMol stays lighter the longer it runs.

### Rendering & export
- **Accurate high-resolution export.** Depth-of-field blur and outline width now scale with the output resolution, so a hi-res render looks like what you see on screen (WYSIWYG).
- **Cleaner tone mapping.** Tone-map and exposure no longer subtly muddy the render.

### Performance
- **Smoother movie export.** Movie rendering now runs off the main thread, so the interface stays responsive while a movie is being written.
- **Snappier color editing.** The inspector color pickers are debounced, so dragging a color no longer stutters the view.

### Under the hood
- AI/MCP control is now thread-safe — automated commands are marshaled onto the main thread, fixing intermittent failures.
- Packaging scripts now fail loudly rather than ever shipping a broken bundle.

Built on the open-source PyMOL engine. Updates install automatically via the in-app updater (**Check for Updates…** in the app menu).
